# Pre-submission Checklist（投稿前自检）

> 目标：减少 desk reject / 减少来回返工。  
> 用法：逐条打勾，并在括号里填“证据路径/链接”。

## A. 期刊/会议硬约束
- [ ] 文章类型与范围匹配（对应 Aims & Scope）
- [ ] 字数 / 图数 / 表数 / 补充材料限制符合
- [ ] 版式模板已套用（LaTeX/Word/Quarto）（证据：`manuscript/`）

## B. 结构与叙事（审稿人视角）
- [ ] 研究问题明确（Introduction 首段能回答：为什么重要）
- [ ] 贡献点明确（contributions 3 条以内，且可验证）
- [ ] Results 与 Figures 一一对应（每张图有一个结论句）
- [ ] Discussion 不重复 Results，只解释意义/局限/未来

## C. 方法可复现
- [ ] 数据来源/采集/纳入排除标准写清楚
- [ ] 统计检验/模型设置/超参/随机种子写清楚
- [ ] 代码与数据可用性声明（可公开/受限/需申请）

## D. 图表（顶刊常见拒稿点）
- [ ] 统一字体/字号/线宽（跨所有图一致）
- [ ] 色盲友好（Okabe-Ito / colorcet / cmcrameri）
- [ ] 导出格式符合：矢量优先（PDF/EPS/SVG），必要时 TIFF 600dpi
- [ ] 图注自解释：样本量、重复次数、统计检验、误差定义（SD/SEM/CI）
- [ ] 子图标注一致：A/B/C（位置、字号、间距统一）

## E. 引用与伦理合规
- [ ] 所有关键主张都有数据或引用支撑
- [ ] 引用格式一致（BibTeX/EndNote/手工都可，但必须一致）
- [ ] 伦理声明/利益冲突/资金来源/作者贡献（按期刊要求）

## F. 最终打包
- [ ] 主文 PDF 可打开、无缺字、无 overfull box（LaTeX）
- [ ] 图片文件可被投稿系统识别（不含奇怪字符/空格）
- [ ] 所有上传文件命名规范（无 `final_v7`）
