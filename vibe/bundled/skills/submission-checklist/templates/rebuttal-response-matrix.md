# Rebuttal Response Matrix（回复审稿意见矩阵）

> 用法：把每条 reviewer comment 拆成“可回答的问题”，逐条回应。  
> 元规则：每条回应必须可追踪到“你改了哪里”。

## 统一写作规范（强制）

1) **先感谢 + 复述问题**：证明你理解对方在问什么  
2) **说明行动**：我们增加了实验 / 改了图 / 补充了分析 / 澄清了方法  
3) **给证据**：新的结果、图编号、表编号、或引用  
4) **定位改动**：章节/页码/行号（至少一个定位信息）

## 禁忌（高频踩雷）

- 不要说“我们不同意”然后不解释  
- 不要把情绪写进 rebuttal（哪怕审稿人很离谱）  
- 不要用“it is obvious”/“clearly”/“obviously”压人  
- 不要引用不存在的结果（审稿人会追问）
